# Git Standards

## 1. Commits
- Small, atomic commits.
- Format:
  - `feat: description`
  - `fix: description`
  - `docs: description`
  - `refactor: description`

## 2. Pull Requests
- Must reference issue/task.
- Require review before merge.
- PR template includes:
  - Summary
  - Screenshots
  - Testing notes

## 3. Code Reviews
- Focus on correctness, readability, performance.
- Not nitpicks unless meaningful.
