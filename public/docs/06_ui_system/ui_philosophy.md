# UI Philosophy

## Purpose
Codify interaction principles for coach-facing UX so implementation remains consistent and minimal-touch.

## Scope
Applies to all program-facing pages, modals, and high-frequency workflows.

## Authoritative statements
- Minimal-touch policy: minimize clicks/steps while keeping screens uncluttered.
- A coach must always know: where they are, where to go, what happens on action, whether it is reversible.
- Prefer large centered modals for deep workflows (e.g., practice planner) with optional inner slide-out panels for advanced detail.
- Prefer drag-and-drop where feasible to reduce keystrokes and cognitive load.

## System interactions
- Component-level contracts should be documented in `interaction_contracts.md`.
